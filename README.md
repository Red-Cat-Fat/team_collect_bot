# Telegram Poll Bot

Бот для группового чата с темами (forum topics), который ежедневно опрашивает участников о посещении игры и тегает тех, кто ещё не ответил.

## Функционал

- Админ устанавливает дату игры командой `/set_game_date DD-MM`
- **Каждый день в 10:00** (с завтрашнего дня после установки даты и до дня игры) бот отправляет/обновляет опрос в заданном топике
- В сообщении опроса три секции:
  - ✅ **Едут** — список проголосовавших «Да»
  - ❌ **Не едут** — список проголосовавших «Нет»
  - ⏳ **Не ответили** — теги участников, которые ещё не голосовали
- Участники голосуют нажатием кнопок «Да»/«Нет»
- Бот мгновенно обновляет сообщение опроса после каждого голоса
- Команда `/who_is_going` — мгновенный вывод списка едущих (только для админов)

## Команды

| Команда | Описание | Права |
|---------|----------|-------|
| `/set_game_date DD-MM` | Установить дату игры (год — текущий или следующий, если дата прошла) | Админ |
| `/who_is_going` | Показать список тех, кто нажал «Да» | Админ |

## Как получить необходимые данные

### 1. BOT_TOKEN
1. Откройте Telegram и найдите **@BotFather**
2. Отправьте команду `/newbot` или выберите существующего бота
3. Следуйте инструкциям: введите имя бота и username (должен оканчиваться на `_bot`)
4. BotFather выдаст токен вида: `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`
5. Скопируйте этот токен — это и есть `BOT_TOKEN`

### 2. ADMIN_IDS
1. В Telegram найдите **@userinfobot** (или @myidbot, @getmyid_bot)
2. Перешлите ему любое сообщение от себя (или нажмите Start)
3. Бот вернёт ваш `Id: 123456789` — это ваш числовой ID
4. Можно указать несколько админов через запятую: `123456789,987654321`

### 3. TOPIC_ID (ID темы/топика)
**Вариант А — через бота:**
1. Добавьте бота **@userinfobot** в вашу группу
2. Включите в группе **Темы (Forum Topics)** — Настройки группы → Темы → Включить
3. Зайдите в нужную тему и перешлите любое сообщение из неё боту @userinfobot
4. Бот покажет `Message thread ID: 123` — это и есть `TOPIC_ID`

**Вариант Б — вручную:**
1. Добавьте своего бота в группу и сделайте админом
2. Напишите в теме любое сообщение
3. Перейдите по ссылке: `https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates`
4. Найдите в JSON поле `message_thread_id` — это `TOPIC_ID`

> ⚠️ **Важно:** Бот должен быть **администратором** в группе (права: отправка сообщений, чтение сообщений, управление темами). Без прав админа бот не сможет получать список участников для тегирования неответивших.

### 4. TIMEZONE
Часовой пояс для планировщика (по умолчанию `Europe/Moscow`).
Список таймзон: https://en.wikipedia.org/wiki/List_of_tz_database_time_zones
Примеры: `Europe/Moscow`, `Europe/Kiev`, `Asia/Yekaterinburg`, `UTC`

## Быстрый запуск (Docker)

```bash
git clone https://github.com/Red-Cat-Fat/team_collect_bot.git
cd team_collect_bot
./run_bot.sh
```

Скрипт `run_bot.sh` сам задаст все вопросы, создаст `.env`, соберёт образ и запустит контейнер.

### Ручной запуск (без скрипта)

1. Скопируйте `.env.example` в `.env` и заполните:
   ```env
   BOT_TOKEN=your_bot_token
   ADMIN_IDS=123456789,987654321
   TOPIC_ID=123
   DB_PATH=./data/bot.db
   TIMEZONE=Europe/Moscow
   ```

2. Создайте папку для БД:
   ```bash
   mkdir -p data
   ```

3. Запустите:
   ```bash
   docker compose up -d --build
   ```

## Логика дат

- Формат: `DD-MM` (например, `15-06`)
- Год берется текущий
- Если дата уже прошла в текущем году — используется следующий год
- **Ежедневно в 10:00** (по таймзоне) бот публикует/обновляет опрос — от установки даты и до дня игры включительно
- В сообщении опроса всегда актуальный список неответивших (теги)

## Структура БД (SQLite)

- `polls` — опросы
- `votes` — голоса пользователей
- `game_schedule` — расписание игр

## Требования

- Docker и Docker Compose v2
- Бот должен быть админом в группе (для отправки сообщений в топик и получения списка участников через `getChatMembers`)
